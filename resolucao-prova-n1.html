Questão 1

De acordo com o Angelotti, qual é a principal razão pela qual a área de banco de dados é considerada de grande importância na informática?

  • a) Porque a informação é um bem precioso e deve ser armazenada de forma coerente e adequada.

Explicação:

A resposta correta é a (a). O cerne da disciplina de Banco de Dados é a gestão da informação. A informação é um ativo valioso para qualquer organização. Bancos de dados garantem que essa informação seja guardada de maneira segura, consistente, organizada e acessível, o que é fundamental para a tomada de decisões e o funcionamento de qualquer sistema.

Questão 2

Na fase de Modelagem Conceitual de um projeto de banco de dados, qual é a principal preocupação?

  • a) Escolher o SGBD que será utilizado e conhecer a linguagem SQL.
  • b) Descrever quais dados serão armazenados e como eles se relacionam, refletindo a necessidade do usuário.
  • c) Transformar o modelo em algo mais próximo da implementação, como o modelo relacional.
  • d) Crie as tabelas, chaves estrangeiras e primárias.
  • e) Otimizar os aplicativos para funcionarem melhor.

Questão 2

  • b) Descrever quais dados serão armazenados e como eles se relacionam, refletindo a necessidade do usuário.

Explicação:

A resposta correta é a (b). A modelagem conceitual é a primeira etapa do projeto e foca em entender e documentar os requisitos de dados do negócio. Ela é de alto nível, independente de tecnologia (SGBD) e descreve as principais entidades (coisas) sobre as quais queremos guardar informação e como elas se associam. O objetivo é criar um mapa que represente o mundo real do usuário.

Questão 3

Uma chave primária (PK) é um atributo ou conjunto de atributos que identifica unicamente uma entidade em uma relação. Qual das seguintes afirmações sobre chaves primárias é INCORRETA?

  • a) Uma chave primária não pode conter valores nulos (NULL).
  • b) Em uma tabela, podem existir múltiplas chaves primárias.
  • c) Ela é escolhida a partir de um conjunto de chaves candidatas, por exemplo Matrícula e CPF.
  • d) Seus valores devem ser únicos para cada registro na tabela, não permitindo repetição.
  • e) Pode ser formada pela combinação de mais de um atributo (chave primária composta).

Questão 3

  • b) Em uma tabela, podem existir múltiplas chaves primárias.

Explicação:

A afirmação incorreta é a (b). Uma tabela (relação) só pode ter UMA chave primária. A função da chave primária é garantir a unicidade de cada registro (linha). Se múltiplas chaves primárias fossem permitidas, este conceito fundamental seria quebrado. Uma tabela pode ter várias chaves candidatas (atributos que poderiam ser a PK), mas apenas uma é escolhida para ser a chave primária.

Questão 4

O Diagrama Entidade-Relacionamento (DER) é uma abordagem para modelagem de dados no nível conceitual. Qual de seus componentes é utilizado para descrever uma associação entre duas ou mais entidades?

  • (a) Entidade.
  • (b) Atributo.
  • (c) Cardinalidade.
  • (d) Grau.
  • (e) Relacionamento.

Questão 4

  • (e) Relacionamento.

Explicação:

A resposta correta é a (e). O componente que descreve uma associação ou interação entre entidades é o Relacionamento, geralmente representado por um losango no DER.
- Entidade: Representa um objeto do mundo real (ex: ALUNO).
- Atributo: Característica de uma entidade (ex: nome do aluno).
- Cardinalidade: Define o número de instâncias de uma entidade que podem se associar a instâncias de outra.
- Grau: Número de entidades que participam do relacionamento.

Questão 5

Considere o seguinte diagrama E-R.

Com base nesse diagrama E-R, é correto afirmar que:

  • a) o atributo código na tabela DEPENDENTE não é uma chave primária.
  • b) na tabela EMPREGADO o atributo código é uma chave estrangeira.
  • c) o atributo nome da tabela DEPENDENTE é uma chave primária.
  • d) o símbolo losango representa um auto-relacionamento entre EMPREGADO e DEPENDENTE.
  • e) a entidade DEPENDENTE só existe porque existe a entidade EMPREGADO.

Questão 5

  • e) a entidade DEPENDENTE só existe porque existe a entidade EMPREGADO.

Explicação:

A resposta correta é a (e). A cardinalidade (1,1) do lado de EMPREGADO no relacionamento "possui" indica que um DEPENDENTE deve estar associado a exatamente um EMPREGADO. Isso significa que a existência de um DEPENDENTE está condicionada à existência de um EMPREGADO, caracterizando uma dependência de existência. Isso também sugere que DEPENDENTE é uma entidade fraca (geralmente representada com um retângulo duplo).

Questão 6

Analise o Diagrama Entidade-Relacionamento de um sistema acadêmico abaixo:

Marque a afirmativa correta sobre o diagrama apresentado:

  • a) Uma disciplina pode ser lecionada por vários professores ao mesmo tempo.
  • b) A entidade TURMA é uma entidade fraca, identificada pela disciplina a que pertence.
  • c) Um professor não pode estar cadastrado no sistema sem lecionar nenhuma disciplina.
  • d) O relacionamento "Leciona" é do tipo Muitos-para-Muitos (N:N).
  • e) A chave primária da entidade TURMA é composta apenas pelo atributo "Ano_Letivo".

Questão 6

  • d) O relacionamento "Leciona" é do tipo Muitos-para-Muitos (N:N).

Explicação:

A resposta correta é a (d). Analisando a cardinalidade do relacionamento "Leciona", vemos (1,n) do lado de PROFESSOR e (0,n) do lado de DISCIPLINA. Isso se lê:
- Um Professor leciona de zero a muitas (n) Disciplinas.
- Uma Disciplina é lecionada por de um a muitos (n) Professores.
Como ambos os lados podem ter "muitos" (n), o relacionamento é do tipo Muitos-para-Muitos (N:N).

Questão 7

No processo de mapeamento de um modelo conceitual para o modelo relacional, a correta representação dos relacionamentos é fundamental. Qual das alternativas a seguir descreve corretamente as regras de conversão para os relacionamentos do tipo 1:1, 1:N e N:N?

  • a) Para relacionamentos 1:N, cria-se uma nova tabela. Para N:N, a chave primária do lado "N" migra para o lado "1". Para 1:1, as tabelas são fundidas em uma só.
  • b) Para relacionamentos 1:N, a chave primária da tabela do lado "N" migra como chave estrangeira para o lado "1". Para N:N, cria-se uma nova tabela. Para 1:1, a chave primária de uma tabela migra para a outra.
  • c) Para relacionamentos 1:N e N:N, o procedimento é o mesmo: cria-se uma nova tabela que armazena as chaves primárias de ambas as entidades como chave estrangeira. Para 1:1, não há necessidade de chave estrangeira.
  • d) Para relacionamentos 1:N, a chave primária do lado "1" migra para o lado "N". Para N:N, não é necessário criar uma nova tabela, apenas adicionar chaves estrangeiras em ambas as tabelas originais. Para 1:1, cria-se sempre uma nova tabela.
  • e) Para relacionamentos 1:1, a chave primária de uma das tabelas migra como chave estrangeira para a outra. Para 1:N, a chave primária da tabela do lado "1" migra como chave estrangeira para a tabela do lado "N". Para N:N, cria-se uma nova tabela, cuja chave primária é composta pelas chaves primárias das entidades envolvidas.

Questão 7

  • e) Para relacionamentos 1:1, a chave primária de uma das tabelas migra como chave estrangeira para a outra. Para 1:N, a chave primária da tabela do lado "1" migra como chave estrangeira para a tabela do lado "N". Para N:N, cria-se uma nova tabela, cuja chave primária é composta pelas chaves primárias das entidades envolvidas.

Explicação:

A resposta correta é a (e), que resume perfeitamente as regras de mapeamento:
- 1:1 (Um-para-Um): A chave primária (PK) de uma das tabelas "migra" para a outra como chave estrangeira (FK). A escolha de qual tabela recebe a FK depende da participação (opcional ou obrigatória).
- 1:N (Um-para-Muitos): A PK da tabela do lado "1" sempre migra para a tabela do lado "N" como FK. Isso é conhecido como "a chave desce".
- N:N (Muitos-para-Muitos): É obrigatório criar uma nova tabela (tabela associativa) para representar o relacionamento. A chave primária dessa nova tabela é, por padrão, a combinação das chaves primárias das duas tabelas originais.

Questão 8

Dado o modelo conceitual de um sistema de blog simples abaixo, escreva o esquema do modelo relacional correspondente. Especifique as tabelas, seus atributos, e identifique as chaves primárias (PK) e estrangeiras (FK).

Questão 8

Esquema Relacional Resultante:

USUARIO(ID_Usuario, Nome, Email)

AUTOR(ID_Autor, Area_interesse, Formacao, ID_Usuario (FK))
    ↳ ID_Usuario referencia USUARIO

POST(ID, Titulo, Texto, Imagens, ID_Autor (FK))
    ↳ ID_Autor referencia AUTOR

COMENTARIO(ID_Comentario, Texto_comentario, ID_Post (FK), ID_Usuario (FK))
    ↳ ID_Post referencia POST
    ↳ ID_Usuario referencia USUARIO

Justificativa do Mapeamento:
• O relacionamento 1:1 "É Um" entre USUARIO e AUTOR foi mapeado com a PK de USUARIO (ID_Usuario) migrando para AUTOR como FK.
• O relacionamento 1:N "Cria" entre AUTOR e POST foi mapeado com a PK de AUTOR (ID_Autor) migrando para POST como FK.
• O relacionamento N:N "Comenta" entre USUARIO e POST gerou uma nova tabela COMENTARIO, que contém as PKs de ambas as tabelas (ID_Usuario e ID_Post) como FKs. Foi adicionada uma PK artificial (ID_Comentario) para a nova tabela.
