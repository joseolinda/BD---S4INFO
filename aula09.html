Linguagem de Definição de Dados (DDL)

Capítulo 04: Criando a Estrutura do Banco de Dados

O que é DDL?

A Linguagem de Definição de Dados (DDL) é um conjunto de instruções SQL que fornece meios para a criação, alteração e exclusão de tabelas e índices.

Criando Tabelas: CREATE TABLE

Sintaxe Básica

Uma tabela é definida usando o comando `CREATE TABLE`. Você deve especificar o nome da tabela, seus campos (colunas) e os tipos de dados de cada campo.

create table T (
    A1 D1,
    A2 D2,
    ...
)
-- Exemplo da Apostila (p. 10)

create table departamento (
    Codigo   decimal(5),
    Nome     char(20)
);

create table funcionario (
    matricula  decimal(5),
    nome       char(30),
    rg         decimal(9),
    sexo       char(1),
    depto      decimal(5),
    endereco   varchar(40),
    cidade     varchar(20),
    salario    decimal(10,2)
);

Restrições: Chave Primária (PRIMARY KEY)

PRIMARY KEY

A Chave Primária (`PRIMARY KEY`) é a restrição mais importante. Sua função é identificar univocamente cada registro (linha) da tabela.

  • Toda tabela deve possuir uma chave primária.
  • Ela pode ser composta por um ou mais campos.
  • Campos da chave primária são implicitamente `NOT NULL` (não nulos).
-- Sintaxe 1: Na definição da coluna
create table departamento (
    Codigo decimal(5) NOT NULL PRIMARY KEY,
    Nome   char(20)
);

-- Sintaxe 2: No final da tabela
-- (Obrigatória para chaves compostas)
create table funcionario (
    matricula decimal(5) NOT NULL,
    nome      char(30),
    ...
    PRIMARY KEY (matricula)
);

Restrições: NOT NULL e UNIQUE

Evitando Valores Nulos

A cláusula `NOT NULL` garante que um campo seja de preenchimento obrigatório. O SGBD apresentará um erro se o usuário tentar inserir um registro sem valor para este campo.

Evitando Valores Duplicados

A cláusula `UNIQUE` garante que o valor armazenado em um campo seja único em toda a tabela. Não podem haver dois registros com o mesmo valor.

create table funcionario (
    matricula decimal(5) NOT NULL PRIMARY KEY,
    
    -- Preenchimento obrigatório
    nome char(30) NOT NULL,
    
    -- Não pode haver RGs duplicados
    rg decimal(9) NOT NULL UNIQUE,
    
    sexo char(1),
    ...
);

Restrições: DEFAULT e CHECK

Valores Padrão (DEFAULT)

A cláusula `DEFAULT` define um valor padrão para um campo. Se o usuário não especificar um valor durante a inserção, este valor padrão será usado automaticamente.

Valores Válidos (CHECK)

A cláusula `CHECK` restringe os valores de um campo a um determinado conjunto ou condição, evitando dados inválidos.

create table funcionario (
    ...
    sexo char(1) CHECK (sexo in ('M', 'F')),
    ...
    -- Valor padrão 'São Paulo'
    cidade varchar(20) DEFAULT 'São Paulo',
    
    -- Salário deve ser maior que 350
    salario decimal(10,2) CHECK (salario > 350)
);

Restrições: Integridade Referencial (FOREIGN KEY)

FOREIGN KEY

A Chave Estrangeira (`FOREIGN KEY`) garante a integridade referencial entre tabelas.

Ela define que o valor em um campo (ex: `depto` em `funcionario`) deve existir na chave primária de outra tabela (ex: `codigo` em `departamento`).

-- Sintaxe 1: Inline
...
depto decimal(5) REFERENCES departamento(codigo),
...

-- Sintaxe 2: No final da tabela
...
FOREIGN KEY (depto) 
    REFERENCES departamento (codigo)
...

Alterando uma Tabela: ALTER TABLE

Adicionar Campos

alter table T
add A1 D1,
add A2 D2, ...

Excluir Campos

alter table T
drop A1,
drop A2, ...

Alterar Campos

alter table T
alter A1 to nA1; (muda nome)

alter table T
alter A1 type t1; (muda tipo)

Removendo uma Tabela: DROP TABLE

DROP TABLE T

CUIDADO!

O comando `DROP TABLE` remove permanentemente a tabela e todas as informações sobre ela do banco de dados.

Isso inclui a estrutura, os dados, os índices e as restrições. A operação não pode ser desfeita facilmente.

Índices: CREATE INDEX

O que são Índices?

Índices são componentes do banco de dados destinados a agilizar o acesso aos dados. Funcionam de forma similar ao índice de um livro, permitindo que o SGBD encontre os registros mais rapidamente.

Podem ser associados a uma ou várias colunas.

-- Cria um índice padrão
create index i_func_rg
    on funcionario (RG);

-- Cria um índice que também força unicidade
create UNIQUE index i_func_rg_unico
    on funcionario (RG);

Gerenciando Índices: Vantagens e Custos

Vantagens

Existem duas razões principais para usar índices:

  • Integridade: Evita dados duplicados, usando a cláusula `UNIQUE`.
  • Desempenho: Aumenta a rapidez das consultas (`SELECT`) que utilizam os campos indexados no `WHERE`.

Cautela e Remoção

Tenha cautela! Índices não são gratuitos. Eles ocupam espaço em disco e podem tornar operações de escrita (`INSERT`, `UPDATE`, `DELETE`) mais lentas, pois o índice também precisa ser atualizado.

Para remover um índice:

-- Remove o índice pelo nome
drop index i_func_rg;
